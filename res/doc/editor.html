The Code Editor

The code editor's behavior depends on the syntax, or type, of code being edited. When a file is opened the file's extension is normally used to determine the syntax for editing its contents. There are currently five supported syntaxes:

Syntax File Types
Lisp .lisp .lsp .cl .scm .cm .clm .ins .cmn .fms .asd
SAL2 .sal2
SAL .sal
Fomus .fms
Text .text .txt .*

If the first line of the file is a special customize comment then the editor will read that as well to configure the window exactly the way the user want.

Command Key Shortcuts

  • On Linux and Windows the Command key is the Control key.
  • On OS X the Command key is the Apple key just to the left of the Spacebar.

On Linux and Windows the command key shortcuts are disabled if Emacs Mode is turned on because Emacs uses the Command key as its Control key. On OS X the Command key is active even when Emacs Mode is turned on.

Emacs Lite Commands

If Edit>Emacs Mode is turned on the code editor supports the Emacs commands listed below. Note that on Windows and Linux the default setting for Emacs mode is off because the Emacs' Control key is the same as Windows' command key

Emacs commands are keystroke combinations involving the CONTROL and/or META keys combined with one (or more) additional keys. In the documentation that follows pressing CONTROL or META is shortened to C- and M- respectively. For example, C-f means to hold down the Control key and type the f key and C-M-f means to hold down both the Control and Meta keys while typing the f key. Note: on OS X the Meta key is the Option key just right of the control key. On Linux and Windows the Meta key is the Alt key just to the left of the Spacebar.
