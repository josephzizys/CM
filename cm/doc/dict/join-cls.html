join

Defines a union of two or more patterns. Elements from the joined patterns are read in parallel and returned as lists in which the elements are in the same order as the patterns specified to the join. The behavior of a join is controlled by the value of its :format initialization.

join supports the following slot initializations:

:format list
Specifies the method for joining data from the individial patterns. If :format is not specified then by default the join is at end-of-period when the all subpatterns ares at end-of-period and each pattern is read every time the join is read. Otherwise :format is a list whose elements correspond in left-to-right order with the patterns to be joined. Each element in list determines if the corresponding pattern controls the period length of the join, or is read one per value or once per period or the join. Possible values are :eop, boolean true, or boolean false. A value of :eop means that the corresponding pattern is read each time the join is read and controls when the join is at end-of-period. If the value is boolean true then the corresponding pattern is read each time the join is read. If the value is boolean false then the corresponding pattern is read only once per period of the join. A false value "spreads" the elements from that pattern over all the elements in each period of the join rather than for each value returned. 
cm> (define a (new cycle :of '(c4 d4) :for 3))

cm> (define b (new random :of '(e q h) :for 2))

cm> (define c (new join :of (list a b)))

cm> (next c true)
((c4 h) (d4 q) (c4 h) (d4 e) (c4 q) (d4 e))
cm> (define a (new cycle :of '(c4 d4 )))

cm> (define b (new random :of '(e q h) ))

cm> (define c (new join :of (list a b) 
                  :format '(:eop #f)))

cm> (next c  true)
((c4 h) (d4 h))
cm> (next c true)
((c4 e) (d4 e))
cm>
