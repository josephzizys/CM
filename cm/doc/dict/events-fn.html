[Function]
(events objects destination [ahead] ...)

Writes events from objects to destination. objects can be a list or a single object. destination can be a file, port, seq or display. ahead is an optional start time offset for objects. If ahead is a single number it is applied to all objects. If ahead is a list then numbers in the list are matched in left to right order with the corresponding objects in objects. Following ahead may come any number of initializations suitable for destination or to its post-processing hooks. events returns destination as its value. 

cm> (events #&foo "test.midi")
"test.midi"
cm> (events (list #&foo #&bar) "test.midi" '(0 2))
"test.midi"
cm> (events #&foo "test.midi" 10 :versions true )
"test-1.midi"
cm>
