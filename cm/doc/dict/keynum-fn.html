[Function]
(keynum freq [:hz bool] [:in tuning] [:in? scale] [:to mode] [:through stale] [:from mode])

Returns the key number of freq, which can be a note, key number, Hertz value or list of the same. For equal tempered tunings, keynum may return a floating point number with a non-zero fractional part. In the standard chromatic scale the fractional value multiplied by 100 is equivalent to cents above the the integer keynum. To return only integer keynums use the :through parameter.

keynum supports the following keyword parameters:

:hz bool
If true then a numerical freq is interpreted as a Hertz value rather than as a key number.
:in tuning
The tuning to use. Defaults to the global variable *scale*. The parameters :in, :in?, :through and :from are mutually exclusive.
:in? {tuning | mode}
Tests if freq references a keynum in the specified tuning or mode. Returns that value or false. The parameters :in, :in?, :through and :from are mutually exclusive.
:to mode
Forces freq to the closer integer keynum in mode and returns that value in modal coordinates. The parameters :in, :in?, :through and :from are mutually exclusive.
:through {tuning | mode}
Filters freq through tuning or mode and returns the closest integer keynum in tuning coodinates. The parameters :in, :in?, :through and :from are mutually exclusive.
:from mode
Converts a key number freq specified in mode's coordinates into a keynum in the mode's tuning. The parameters :in, :in?, :through and :from are mutually exclusive.
from or to its equivalent position in the specified scales. 

cm> (keynum 'c4)
60
cm> (keynum 440 :hertz)
69.0
cm> (keynum 452.893 :hz)
69.5
cm> (keynum '(cs5 ds fs gs as cs6))
(73 75 78 80 82 85)
cm>
