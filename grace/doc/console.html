Overview:
The Console Window

The Console window serves as the main window for Grace: there is only one Console window and closing it will quit the application. The purpose of the Console is to display messages, errors and warndings produced by Grace commands. It also serves as standard ouput for Lisp.

Colort Themes

The appearance of the Console can be customized by applying color themes to it. A number of different themes are available under the View>Themes sub menu. In general, a theme controls the Console's background color and the color of normal output text.

A few special types of messages have a consistent appearance across all themes:

  • Error/fail messages (red)
  • Warning messages (orange)
  • Return/success values (green)

The Lisp Menu

The Lisp menu allows you configure, start, stop and customize aspects of a Lisp session.

Configure Lisp Dialog

Use the Lisp>Configure Lisp... dialog window to specify the information that is required to start a Lisp session. Here is an explanation of the various fields of this dialog:

  • The Host field allows you to specify the computer that will act as the "Lisp server" that Grace will connect to and communicate with. Currently this field "read only" and is set to localhost, i.e. the physical machine you are using.
  • The Port field allows you to specify the port (socket) number that will be used to communicate with Lisp once the Lisp server has started running. The port number can be any value above 1024. Normally there is no reason to change the default port value. However, if you have already attempted to start Lisp and failed you might bump this number to insure that a new port will be used for the next connection.
  • The radio buttons allow you to select the Lisp implementation that you will use during the Lisp session. Grace supports either SBCL or CLisp. When you select one of the implementations, the default path to that lisp's executable will be displayed in the typein field. If the default location is not correct (or if none is supplied) then either type the absolute pathanme to the exectuabtle or use the browse button next to it to locate the executable using a Find File dialog.
  • The ASDF Systems Directory field contains the pathname to the directory on your machine where you store the various Lisp systems you work with. Add any Lisp ASDF system you want to this directory for easy access and loading. Since ASDF systems get compiled when they are loaded, you must have write permission in these directories.

Starting and Stopping Lisp

If the Lisp>Start Lisp item is active select it to start a Lisp session. If it is not active then use Lisp>Configure Lisp... to configure the session so that it can be started. If Lisp>Stop Lisp is available you can use it to kill the running Lisp session.

Grace will print information as it attempts to start your Lisp and establish a socket connection with it. If the connection is successful your Lisp session is ready to go. If the connection is not successful then use the Configure Lisp dialog to fix the problem and use Start Lisp again to attempt another connection.

Once Lisp is running the Console window will display output that results from the Lisp session. Note that the Console window is not a Lisp REPL (this is a feature not a bug): to evaluate Lisp or SAL code open a new Editor window and enter your code there. Then place your cursor at the end of the text to execute and press COMMAND-Enter, or use the Execute menu item under the SAL or Lisp menu.
